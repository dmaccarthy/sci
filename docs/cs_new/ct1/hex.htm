Hexadecimal

In computer programming, it is common to use a base 16 number system called hexadecimal.

  • We have already learned that binary (base 2) uses two symbols (0 and 1) and the place value of each digit is two times as big as the next digit (1, 2, 4, 8, 16, ...)
  • Binary numbers are often prefixed with 0b to allow compilers and interpreters (like Python or Javascript) to tell them apart from regular decimal numbers.
  • Decimal (base 10) uses ten symbols (0...9) and the place value of each digit is ten times as big as the next digit (1, 10, 100, 1000, ...)

  • Hexadecimal numbers are often prefixed with 0x to distinguish them from decimal numbers.
  • The place value of each digit is sixteen times larger than the next digit!

Suppose we encountered a hexadecimal number 0xA3E...

Hex DigitsA = 103E = 14
Place Value162 = 256161
Value10 × 256 = 25603 × 16 = 4814
  • The last digit (E) is in the 1’s column and is equivalent to the decimal number 14.
  • The next digit (3) is in the 16’s column, so it is counted as 3 × 16 which is 48.
  • The next digit (A) is in the 256’s column and is equivalent to the decimal number 10, so it is counted as 10 × 256 which is 2560.
  • The decimal equivalent of this number is 2622...
print(0xA3E)    # 2622 = 10 * 256 + 3 * 16 + 14
print(0x7D2)    # 2002 = 7 * 256 + 13 * 16 + 2
  • If we add a fourth digit, the place value of the largest column will be 163, or 4096.

The reason that hexadecimal is popular in computer programming is that each hexadecimal digit requires four bits, or one-half of an 8-bit byte for storage. Suppose we have a byte containing the binary number 0b11001001. We can separate the byte into two halves of four bits each...

0b1100 = 8 + 4 = 12
0b1001 = 8 + 1 = 9

We can represent the byte in hexadecimal as 0xC9, where the C (12) represents the first four bits and the 9 represents the last four bits.

To convert a decimal number to hexadecimal, we can:

  • Write the number using powers of 16, or
  • Convert the number to binary, and then encode every group of four bits as a hex digit.

Use the Python format code x to inidicate a hexidecimal number...

dec = 12351
print(f"0x{dec:x}")    # 0x303F = 3 * (16**3) + 3 * (16) + 15 * (1)
print(f"0b{dec:b}")    # 0b11000000111111

Notice that in the binary encoding, the groups of four bits shown by the red and blue digits encode to 3, 0, 3, and 15(F)!!
