XML Applications

The XML format is used to store data for many common applications, including:

  • Office Documents: Many Office programs such as Microsoft Office (Word, Excel, etc.), LibreOffice and OpenOffice use XML to save document information and content.
  • Scalable Vector Graphics: Most graphics formats (jpeg, png, bmp) store images as an array of pixels. When the image is resized it creates distortion. SVG uses XML to describe the image geometrically instead of as individual pixels. SVG images can be resized without loss of image quality.
  • MathML & SMIL: These are XML applications for describing mathematics and multimedia content respectively.
  • GeoGebra: This is a geometry and algebra program that stores data using XML. This application can be used to embed dynamic drawings in webpages.

Hypertext Markup Language

The World Wide Web was created in the early 1990’s with the development of the Hypertext Markup Language. HTML is a markup language that describes documents (webpages) available for viewing over the Internet. HTML was based on (but not completely compatible with) an earlier markup language known as Standard Generalized Markup Language (SGML) which was formalized in 1986.

XML is actually a subset of SGML. The first XML specification was published in 1998. XML and HTML are similar as a result of their shared SGML ancestry. Once the XML specification was released, work began on revising HTML to make it fully compatible with XML. The result, in 2000, was the eXtensible Hypertext Markup Language (XHTML): a language for encoding web pages using XML.

More recently, the newer HTML5 language was created. HTML5 is similar to, but not completely compatible with XML. Below is an HTML5 document that describes a blank webpage.

<!DOCTYPE html>
<html lang="en-CA">
  <head>
    <title>Blank</title>
  </head>
  <body>
  </body>
</html>

The root element node in an HTML5 document uses the tag <html> in lower case. It often contains a lang attribute to specify the language in which the document is written. The <!DOCTYPE> declaration is also html.

The <html> element has two child elements, <head> and <body>. The <head> element has a child element <title> which contains the document title as a text node.

The content of the document will be placed in the <body>. The <head> can contain other data such as style information to alter the layout and appearance of the document.

meta Tags

HTML documents may contain one or more <meta> elements in the <head> of the document...

<head>
  <title>Example</title>
  <meta charset="utf-8" />
</head>

The <meta> tag is used to include information about your webpage that can be used by browsers, search engines, and other programs that read the document. In the example above, the charset meta tag identifies the encoding, so that the browser can decode the bytes that make up the HTML file into characters. When creating HTML documents, make sure that you save your files using the same encoding that you have declared in your meta tag.

Paragraphs

The simplest kind of content that you can include in your webpage is text. In HTML, text is added using a <p> tag, which is short for paragraph. Paragraph elements are included as child nodes of the <body> element, and the text content is a child text node of the <p> element. As you can see in the example below, XML comments are supported in HTML...

<!DOCTYPE html>
<html lang="en-CA">
<head>
    <title>Poem</title>
</head>
<body>
    <p><!-- Verse 1 -->
        Mary had a little lamb,
        little lamb, little lamb,
        Mary had a little lamb,
        whose fleece was white as snow.
    </p>
    <p><!-- Verse 2 -->
        And everywhere that Mary went,
        Mary went, Mary went,
        and everywhere that Mary went,
        the lamb was sure to go.
    </p>
</body>
</html>

When you preview the HTML in your browser, it does not display the data tree like it would for an XML file. The web browser recognizes the file as HTML and displays the content as a webpage. If you want to see the HTML markup, activate the browser's “developer tools” (usually by pressing F12), and going to the “Elements” or “Inspector” tab.

The blue text is the content that will be displayed by the browser with the tags organizing the information for the browser.

For our poem webpage, something appears to be wrong! The lines of each verse run together as a single line! This is because web browsers ignore whitespace in HTML. Whitespace means space characters, tab characters, or end-of-line characters. Web browsers treat all whitespace as if it were just a single space. As a result, even though our HTML document has each verse of the poem written as four lines, the poem is rendered with each verse as a single line.

One option is to make each line into its own paragraph. If we did this, the browser would add space between every line. The other option is to use line break (<br/>) tags as shown below.

Note that we have only shown the <body> portion of the document. The <html> and <head> elements shown in black in the previous example must also be included.

<body>
   <p><!-- Verse 1 -->
      Mary had a little lamb,<br/>
      little lamb, little lamb,<br/>
      Mary had a little lamb,<br/>
      whose fleece was white as snow.
   </p>
   <p><!-- Verse 2 -->
      And everywhere that Mary went,<br/>
      Mary went, Mary went,<br/>
      and everywhere that Mary went,<br/>
      the lamb was sure to go.
   </p>
</body>

Headings

HTML provides six levels of heading tags (<h1>, <h2>..<h6>) that work just like the paragraph tag, except that they are used for titles, subtitles, and section headings rather than regular text. Text in a heading tag is displayed more emphatically that text in a normal paragraph; <h1> is the most emphatic, while <h6> is the least emphatic heading.

The examples below only show the <body>. A proper HTML document would also include the <html>, <head>, and <title> tags.

<body>
  <h1>Heading 1</h1>
  <h2>Heading 2</h2>
  <h3>Heading 3</h3>
  <h4>Heading 4</h4>
  <h5>Heading 5</h5>
  <h6>Heading 6</h6>
  <p>Regular Paragraph</p>
</body>

Here is out poem with the title as a heading...

<body>
  <h1>Mary Had a Little Lamb</h1>
   <p><!-- Verse 1 -->
      Mary had a little lamb,<br/>
      little lamb, little lamb,<br/>
      Mary had a little lamb,<br/>
      whose fleece was white as snow.
   </p>
   <p><!-- Verse 2 -->
      And everywhere that Mary went,<br/>
      Mary went, Mary went,<br/>
      and everywhere that Mary went,<br/>
      the lamb was sure to go.
   </p>
</body>
