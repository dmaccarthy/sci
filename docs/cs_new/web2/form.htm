Forms

So far, we have only learned about static “Web 1.0” webpages. The user cannot interact with the webpage except by clicking a link to move to a different page. One way to increase the interactivity of a page is to include forms.

A form is one or more inputs that allows the user to enter information or choose between options. When the user “submits” the form, the page responds by doing something. For example, when you submit an order form to an online bookstore, it causes a program to run on the server that records the order, bills your account, sends a copy of the order to the bookstore’s shipping department, and then sends you a confirmation of your order.

A simpler example of a form is a login page. The form below consists of three inputs: the username is a text input, the password is a password input and the login button is a submit input.

Username: Password:

The HTML code for the form is:

<form>
  Username: <input type="text" name="user" value="guest"/>
  Password: <input type="password" name="pass"/>
  <input type="submit" value="Login"/>
</form>

Each input is a self-closing tag. The type attribute is used to specify the kind of input you want.

  • text: A rectangular area where the user can type text.
  • password: The same as text, except that the characters are hidden as dots or asterisks.
  • submit: A button that submits the form data to the server.

The name attribute identifies the input. Giving each input a name allows the server to tell which data goes with which input. The submit input does not need a name.

The value attribute specifies the default text to displayed for the text and password inputs, and the message displayed on the button for the submit input.

The input elements and their labels need to be nested inside a form element. The form element usually has an action attribute that identifies the URL where the form data is to be sent when the submit button is clicked. The form may also have a method="post" attribute, which submits the form more securely than the default method. Whether a form needs to use the post method or the default method is determined by the server to which it is being submitted.

<form action="https://dmaccarthy.vercel.app/formecho" method="post">
  Username: <input type="text" name="user" value="guest"/>
  Password: <input type="password" name="pass"/>
  <input type="submit" value="Login"/>
</form>

Other Inputs

Forms can also include other inputs such as check boxes, radio buttons, selects, and file uploads.

A check box is an input that can be checked or unchecked. Radio buttons are a group of inputs with the same name attribute, from which the user can check only one. Checking one radio button unchecks all of the others that have the same name.

A select input creates a menu with several options to choose from.

Here is the HTML code for the form above:

<form action="https://dmaccarthy.vercel.app/formecho" method="post" enctype="multipart/form-data">
  <p><input type="checkbox" name="student" value="yes" /> Current Student</p>
  <p>
    <input type="radio" name="grade" value="10" checked="checked" /> Grade 10<br/>
    <input type="radio" name="grade" value="11" />Grade 11<br/>
    <input type="radio" name="grade" value="12" />Grade 12
  </p>
  <p>Residence: <select name="res">
    <option value="ShPk">Sherwood Park</option>
    <option value="Fort">Fort Saskatchewan</option>
    <option value="Ard">Ardrossan</option>
    <option value="County">Strathcona County</option>
    <option value="Edm">Edmonton</option>
    <option value="Other">Other</option>
  </select></p>
  <p>Application Form: <input type="file" name="file"/></p>
  <p><input type="submit" value="Send"/></p>
</form>

The file input allows the user to select a file from their computer and upload it to the server. To permit the file to be uploaded, you have to add an additional attribute called enctype (encoding type) to the form element:

enctype="multipart/form-data"

Without this extra attribute, the browser will only submit the file name, not the actual contents of the file. (You can edit the form above to remove this attribute and see how the server responds differently when you submit the form.)
