Positional Arguments

One useful application of iterable data structures like tuples and dictionaries is that they allow us to write functions that can accept a non-fixed number of arguments. The function below has two parameters, and therefore it must be called with two arguments...

def add(a, b):
    return a + b

print(add(1, 2, 3, 4))
# TypeError: add() takes exactly 2 positional arguments (4 given)

Arguments passed to a function are paired with the function’s parameters by position or order. These are known as positional arguments. In the program above, Python assigns the value 1 to the parameter a and 2 to parameter b. The exception occurs because the interpreter does not know what to do with the additional arguments 3 and 4.

To allow a function to accept “extra” arguments, you can declare a final parameter with its name preceded by an asterisk. By convention, this parameter is usually named args.

def add(a, b, *args):
    print("Extra:", args)      # Extra: (3, 4)
    return a + b

print(add(1, 2, 3, 4))         # 1 + 2 = 3

This time, no error occurs. As before, Python assigns the values 1 and 2 to the positional parameters a and b. The extra arguments 3 and 4 are packed into a tuple. This tuple is passed to the parameter args. The asterisk preceding the parameter name args indicates to the interpreter that this parameter will receive the “extra” arguments (if any) that are passed to the function. We can now call the function add with as many arguments as we want. There will still be a minimum of two required arguments for the required positional parameters. Here is a more practical* version of the add function...

def add(*args):
    "Calculate the total of an arbitrary number of values"
    total = 0
    for x in args:
        total += x
    return total

print(add(1, 2, 3, 4))

(*Practical, but “totally” unnecessary, since Python has a built-in function sum that does the same thing.)

Keyword Arguments

Python also permits functions to accept arguments identified by a keyword rather than by position...

def add(a, b, *args, **kwargs):
    print("Positional:", args)
    print("Keyword:", kwargs)
    return a + b

name = input("Please enter your name: ")
total = add(1, 2, 3, 4, user=name, year=2021)
print(total)

In this example, Python assigns the first two arguments (1 and 2) to the parameters a and b. The remaining arguments without keywords (3 and 4) are packed into a tuple and assigned to the parameter identified by a single asterisk. The keyword arguments are packed into a dictionary and assigned to the parameter identified by a double asterisk. Conventionally, the dictionary parameter is named kwargs (keyword arguments).

When a function is called, the order of the arguments must be: required arguments first, non-keyword arguments next, and keyword arguments last. The following will produce an exception because a non-keyword argument (4) is passed after a keyword argument (user=name)...

total = add(1,2,3, user=name, 4, year=2021)

    a^2 = c^2 - b^2

    K = C + 273.15

    The formulas for converting to Celsius are:

    C = K - 273.15

    C = (F - 32) / 1.8

    Complete the program d04_temp.py so that the temperature function will accept the temperature in any of the units, using keywords C, K, or F. The function should return a dictionary that contains the temperature in all three units.
