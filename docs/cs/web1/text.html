Text Nodes

XML supports several other types of nodes in addition to element nodes. As we have already learned, an element node is represented by a tag that contains a tag name (mandatory), and attributes* (optional). A self-closing tag can be used for elements that have no child nodes, while paired start- and end-tags must be used if the element does have child nodes. (*Technically, attributes are considered to be a node type separate from the element nodes that contain them! However, we are not going to pursue this distinction at this stage of instruction.)

Another type of node that can be included in XML documents is a text node. As the name suggests, text nodes contain only text. Text nodes do not use tags, and therefore have no tag name, no attributes, and no child nodes. Consider the two Country element nodes in this examples...

<Examples>
  <Country name="Canada" capital="Ottawa" />
  <Country capital="Ottawa">Canada</Country>
</Examples>

Both examples are valid XML, and both contain (almost) the same information. The difference is that the second example includes the country name (“Canada”) as a child text node, while the first example includes it as an attribute of the element node instead. When data is placed in a text node, it does not need to be enclosed in quotes like attributes do. The XML parser will recognize text data by the fact that it is not part of a tag.

Of these two methods of encoding the data, the first example is better, because it tells the reader explicitly that “Canada” is the name of the Country. A human reader would likely be able to figure this out for the second example too, but computer programs only know what is actually encoded in the computer’s memory. You should use attributes to encode small quantities of text data (i.e. text that is much smaller than a single line on the computer monitor) while using text nodes for larger quantities, such as a complete paragraph from a novel or essay...

<Novel title="A Christmas Carol" subtitle="A Ghost Story of Christmas" author="Charles Dickens" year="1843">
    <Chapter number="Stave I" title="Marley's Ghost">
        <Paragraph number="1">Marley was dead: to begin with. There is no doubt whatever about that. The register of his burial was signed by the clergyman, the clerk, the undertaker, and the chief mourner. Scrooge signed it: and Scrooge's name was good upon 'Change, for anything he chose to put his hand to. Old Marley was as dead as a door-nail.</Paragraph>
        <Paragraph number="2">Mind! I don't mean to say that I know, of my own knowledge, what there is particularly dead about a door-nail. I might have been inclined, myself, to regard a coffin-nail as the deadest piece of ironmongery in the trade. But the wisdom of our ancestors is in the simile; and my unhallowed hands shall not disturb it, or the Country's done for. You will therefore permit me to repeat, emphatically, that Marley was as dead as a door-nail.</Paragraph>
        <!-- More paragraphs go here... -->
    </Chapter>
    <!-- More chapters go here... -->
</Novel>

Comment Nodes

Another feature of XML is comment node. In computer programming, a “comment” refers to text that is intended only for human readers and is to be ignored by programs that read the code. For example, you might want to explain something when the purpose of the source code isn’t clear, or you might want to make notes to yourself while you are writing your source code without having your notes cause the program to behave in an undesirable way.

In XML, comments are enclosed in tags just as element nodes are. Instead of using a tag name, a special sequence of symbols is used to identify comments...

<ElementNode data="Some Text">
    This is a text node!
    <!-- This is a comment node -->
</ElementNode >
<!-- This is another comment node -->

Note that the characters !-- appear immediately after the less than symbol (<) to identify the beginning of the comment’s tag, and the characters -- appear at the end of the tag, immediately before the greater than symbol (>).
