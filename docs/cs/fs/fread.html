“Reading” a Text File

A text file is a file containing data encoded using a text encoding scheme such as ASCII or UTF-8. That is, the sequence of binary data (0’s and 1’s) contained in the file represents a sequence of characters. Python provides a built-in function named open that you can use to access text files...

# Example 1...

text = open("myData.txt", encoding="UTF-8")
for line in text:
    print(line.rstrip())
text.close()

The first argument of the open function is the absolute or relative path of the file to be opened. The keyword argument encoding identifies the codec that Python is to use to decode the information in the file. You should use the "UTF-8" codec most of the time.

When you open a file, Python and/or your operating system handle much of the work involved in accessing the data such as buffering (deciding when and how much of the file data to load into RAM) and decoding (converting the binary data into str instances).

In Python, text files are iterable. You can loop through an open file with each iteration operating on a single line of text. Text files use a special character or series of characters to separate one line from the next. When Python reads a line of text, the end-of-line character(s) are included as part of the str object produced. In the example above, the rstrip method is used to remove the end-of-line characters (and any other trailing whitespace) from the str instances.

When you have finished with a file, you must call the close method to free up any resources being used to access the file.

Python’s with-as Statement

Python provides a special with-as statement for objects (like files) that have a “clean-up” procedure. In the case of file instances, the clean-up procedure is to close the file.

# Example 2...

with open("myData.txt", encoding="UTF-8") as text:
    for line in text:
        print(line.rstrip())

As in the previous example, the file instance is created with the open function and assigned a name (text). However, rather than using a normal assignment statement, the with-as statement is used instead. When Python reaches the end of the indented block following the with-as statement it runs the instance’s clean-up procedure (i.e. close the file) automatically.

You should use the with-as statement whenever you can with open to ensure that you do not accidentally forget to close your files. This will also close the file if an exception occurs while reading the file.

80's Movie#Raiders of the Lost Ark

[("80's Movie", "Raiders of the Lost Ark"),
    ("Sports Team", "Toronto Raptors"),
    ("Canadian City", "Halifax, Nova Scotia")]
