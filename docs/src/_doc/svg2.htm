Introduction

SVG2 is a lightweight library for making SVG drawings and animations using client-side Javascript.

This tutorial assumes you are already familiar with HTML5, CSS, Javascript, and jquery!

Creating an SVG2 Drawing

Make sure you have included all required jquery and SVG2 scripts in yout HTML5 document.

<script type="text/javascript" src="https://code.jquery.com/jquery-3.7.1.min.js"></script>
<script type="text/javascript" src="src/util.js"></script>
<script type="text/javascript" src="src/math.js"></script>
<script type="text/javascript" src="src/svg2.js"></script>

Create an <svg> tag somewhere in the body of your document:

<p><svg id="Drawing1"></svg></p>

In your Javascript, create an instance of the SVG2 class and associate it with the <svg> tag:

function draw() {
    let svg = new SVG2("#Drawing1", {
        lrbt: [-5, 5, -5, 5],
        scale: 32,
        margin: 2,
        grid: 1
    });

    // Add drawing instructions here later

}

The first argument to the SVG2 constructor is a selector that identifies the <svg> tag to use. In this example, we are using the id value "Drawing1".

The second argument is an object describing the SVG2 instance’s configuration.

Alternatively, if you do not want to use an existing <svg> tag for your drawing, you can call the static function SVG2.create to create a new element, and then add it to your document:

function draw() {
    let svg = SVG2.create({
        lrbt: [-5, 5, -5, 5],
        scale: 32,
        margin: 2,
        grid: 1
    });

    $("body").append(svg.element);

    // Add drawing instructions here later

}

The items in the configuration object in these examples are as follows:

  • lrbt: This stands for “left, right, bottom, top”. It is a 4-array specifying the coordinate system your drawing will use. The first two values assign coordinates to the left and right edges of the drawing and the last two to the bottom and top.
  • scale: An integer that specifies the drawing scale. In this example, 32 means that one unit in the coordinate system equals 32 pixels on the screen. You can also pass a 2-array if you want to use different scales in the x and y dimensions. You can omit the scale option and provided a size option instead. The scale will be calculated so that the drawing size, including margins, will equal the specified size.
  • margin: The number of pixels to add outside of the coordinates specified by lrbt. This can be a 4-array to specify different margins along the left, right, bottom, and top edges.
  • grid: This option causes a coordinate grid to be drawn at the specified interval. This can be a 2-array to have different grid intervals in the x and y dimensions.

Additional options not shown in the example above include:

  • noAxes: By default, the x and y axes are styled differently than other gridlines. Set this option to true to disable this default.
  • appendAxes: By default, the lines representing the axes are rendered last, so that the axes appear on top of the gridlines. Set this to false to disable this default.

Adding Shapes

After creating your SVG2 instance as in the previous example, you can draw basic shapes by adding lines like the following to your draw function:

svg.line([3, 1], [-4, 2]);
svg.circle(2, [3, 1]);
svg.ellipse([1, "25"], [-4, 2]);
svg.rect([2, "20"], [-3, -2]);
svg.poly([[0, 0], [-2, 4], [1, 3]]);

The drawing will now look like this:

The line method draws a line between the two points provided. However, the line will not be visible on the drawing yet because we have not specified the stroke colour.

The circle method draws a circle. The first argument (2) is the radius of the circle and the second argument ([3, 1]) gives the coordinates of the circle’s centre. We will discuss later how to anchor shapes to points other than their centres.

If the drawing uses a different scale for the x and y axes, the size of the circle will be based on an average of the scales. For example, a scale of 32 would mean that a radius of 2 units equals 64 pixels. If the other axis has a scale of 20, 2 units would be 40 pixels. SVG2 calculates a “root mean square average” of 64 and 40, and draws the circle with a radius of about 53 pixels.

By default, a new <circle> element is appended to the <svg> element, however you can pass as a third argument an existing circle to be resized or recentred. This argument can be an SVGCircleElement instance, or a jquery collection or selector string representing the element.

The circle method returns a jquery collection containing the <circle> element.

The ellipse method is similar to the circle method except that the radius argument is replaced by a 2-array specifying different horizontal and vertical radii.

The rect method draws a rectangle. The first argument gives the dimensions of the rectangle and the second the coordinates of its centre. Note that the dimensions in the example are given as [2, "20"]. The unquoted value 2 indicates a width of 2 units in the drawing’s coordinate system. The quoted "20" indicates a height of 20 pixels. SVG2 interprets actual numbers as coordinate system values and numbers-as-strings as pixel values.

The poly method draws either a closed <polygon> or open <polyline>. The first argument is an array of points to connect. The optional second argument is a boolean value specifying whether to draw a closed (true) or open (false, default) shape. Alternatively, the second argument can describe an existing <polygon> or <polyline> element to be modified.

These methods can be called on the SVG2 instance or any group instance to create basic shapes that correspond to a single element in the SVG drawing:

  • circle
  • ellipse
  • image
  • line
  • mjax / mj
  • path
  • poly
  • rect
  • star
  • text

These methods produce composite shapes. They typically return a group which may contain more than one element:

  • arrow
  • chevron
  • coil
  • cylinder
  • edot
  • energy_flow
  • error_bar_x
  • error_bar_y
  • graph
  • label

There are also static functions that create an entire drawing and return a new SVG2 instance...

Styling the Shapes

Since the line, circle, ellipse, rect, and poly methods all return jquery collections, you can use the jquery.css method to set the fill, stroke and stroke-width style attributes.

svg.line([3, 1], [-4, 2]).css({"stroke": "black", "stroke-width": "2px"});
svg.circle(2, [3, 1]).css({"fill": "red", "stroke": "black", "stroke-width": "2px"});
svg.ellipse([1, "25"], [-4, 2]).css({"fill": "#0065fe", "stroke": "none"});
svg.rect([2, "20"], [-3, -2]).css({"fill": "none", "stroke": "red", "stroke-width": "4px"});
svg.poly([[0, 0], [-2, 4], [1, 3]]).css({"fill": "none", "stroke": "limegreen", "stroke-width": "1px"});

The drawing will now look like this:

SVG2 provides a static function style that makes styling more concise. The following code produces the exact same result:

SVG2.style(svg.line([3, 1], [-4, 2]), "black@2");
SVG2.style(svg.circle(2, [3, 1]), "red", "black@2");
SVG2.style(svg.ellipse([1, "25"], [-4, 2]), "#0065fe", "none@");
SVG2.style(svg.rect([2, "20"], [-3, -2]), "none", "red@4");
SVG2.style(svg.poly([[0, 0], [-2, 4], [1, 3]]), "none", "limegreen@1");

The first argument is the jquery collection to be styled. The remaining arguments are style descriptors. The function interprets a string containing the @ symbol as a stroke and stroke-width value; "red@4" is short for {"stroke": "red", "stroke-width": "4px"}. If the string begins or ends with @, it will set only the stroke (e.g. none@) or only the stroke-width (e.g. @2)

Strings that do not contain the @ symbol are taken as fill values.

The SVG2.style function returns the jquery collection that was passed as the first argument.

Grouping & Transforming

The SVG <g> element is used to group items. You can apply styles to a group, which are then inherited by the contents of the group rather than having to style each item individually. You can also use groups to translate, rotate, and animate parts of the drawing.

let blue = svg.group("black@2", "#0065fe", {"fill-opacity": 0.5});
blue.line([3, 1], [-4, 2]);
blue.circle(2, [3, 1]);
blue.ellipse([1, "25"], [-4, 2]);
blue.rect([2, "20"], [-3, -2]);
SVG2.style(blue.poly([[0, 0], [-2, 4], [1, 3]]), "none", "red@2");

In this example, we create a group blue by calling the svg.group method, which appends a new <g> element to the <svg>. The argument list contains the style information to be applied to the group. We then create our shapes by running the circle, rect and other methods on the group identified by blue rather than on svg, to append new elements to the <g> element.

Groups have a config method that we can use to shift and rotate the group. Adding the following line will rotate the entire group by 45° counterclockwise using the coordinates [3, 1] (the centre of the circle) as the pivot point, and then translate the entire group one unit to the left, and one unit up. Theta (θ) is the Greek letter often used to represent angles in mathematics.

blue.config({theta: 45, pivot: [3, 1], shift: [-1, 1]});

If we later change the pivot property, the coordinates we provide are relative to the original, unshifted and unrotated state.

Animations

Here’s a simple animation of Earth and Venus orbiting the Sun. Below that is the function that creates the animation using SVG2. We will go through the function line by line to see how it works.

function draw() {
    // Create the animation
    let svg = new SVG2("#Orbits", {lrbt: [-1.1, 1.1, -1.1, 1.1], scale: 160});

    // Draw the Sun at the origin
    SVG2.style(svg.circle("12"), "yellow");

    // Define a function to make random angles
    let angle = () => 360 * Math.random();

    // Draw and animate Earth
    let earth = svg.group("#0065fe");
    earth.circle("7", [1, 0]);
    earth.config({animated: true, theta: angle(), omega: 20});

    // Draw and animate Venus
    let venus = svg.group("lightgrey");
    venus.circle("6", [0.72, 0]);
    venus.config({animated: true, theta: angle(), omega: 32});

    // Set background to black and pause/resume on mouse click
    svg.$.css("background-color", "black").on("click", () => svg.toggle());

    // Start the animation
    svg.play();
}

The first line creates the animation to run in an <svg> identified by id = "Orbits":

let svg = new SVG2("#Orbits", {lrbt: [-1.1, 1.1, -1.1, 1.1], scale: 160})

We want the Earth to orbit at a radius of 1, so we’ve defined the coordinate system to go a little past 1 on both the positive and negative sides of both axes. The scale sets the overall size of the animation.

Next we draw a yellow circle with a radius of 12 pixels to represent the Sun. We do not need to specify the coordinates of the centre since the default position will be the origin.

SVG2.style(svg.circle("12"), "yellow")

Now we define a function that chooses a random angle between 0 and 360:

let angle = () => 360 * Math.random()

Earth will be drawn as a light blue circle with a radius of 7 pixels. In SVG2, animation is done on <g> elements, so we will create a group, and then add the circle to it:

let earth = svg.group("#0065fe");
earth.circle("7", [1, 0]);

The circle is positioned 1 unit away from the origin so that it will orbit with the desired radius of 1. The animation is accomplished with the next line:

earth.config({animated: true, theta: angle(), omega: 20})

We call the group’s config method to set its animated property to true. This signals to SVG2 that it needs to animate the group when the animation is playing. We use our angle function to rotate the Earth to a random angle. We do not need to specify a pivot point because this defaults to the origin.

Omega (ω) is a Greek letter often used to represent angular speed. Setting this to 20 means that when the animation is running, the theta value will automatically change at a rate of 20° per second.

We repeat these last three lines for Venus, changing the fill colour, circle size, orbital radius, and angular speed:

let venus = svg.group("lightgrey");
venus.circle("6", [0.72, 0]);
venus.config({animated: true, omega: 32, theta: angle()});

We now use jquery to give our <svg> element a black background, and an event listener for mouse clicks.

svg.$.css("background-color", "black").on("click", () => svg.toggle())

The expression svg.$ refers to a jquery collection containing only our <svg> element. We are handling mouse click events by running svg.toggle(), which switches the animation between playing and paused.

The final line automatically starts the animation when the draw function finishes:

svg.play()

In addition to the shift, theta and omega properties, motion of animated groups can be controlled with the vel (velocity), acc (acceleration) and alpha (angular acceleration) properties. The shift, vel, and acc values are 2-arrays specifying both x and y components.
